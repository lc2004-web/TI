## Project Overview

This is a bare-metal embedded project for the **TI MSPM0G3507** microcontroller (ARM Cortex-M0+, thumbv6m). It was generated from TI's "empty" DriverLib example template and uses **Code Composer Studio** (CCS Theia 70.5.1) as the IDE.

## Build & Flash

- **Build**: Use CCS IDE. The project builds via Eclipse CDT Managed Build with GNU Make and the TI ARM Clang Compiler (TICLANG 4.0.4.LTS). Output is `Debug/TEST.out` (ELF).
- **Flash/Debug**: Debug probe is **SEGGER J-Link** (configured in `targetConfigs/MSPM0G3507.ccxml`). Launch debug from CCS.
- **Clean**: `make clean` in `Debug/`, or use CCS clean command.

## Architecture

- **`empty.c`** — Application entry point. Calls `SYSCFG_DL_init()` then runs a while(1) loop toggling the LED on GPIOA.0.
- **`empty.syscfg`** — TI SysConfig file. Defines peripheral/pin assignments (LED on PA0, DEBUGSS on PA19/PA20). All DriverLib init code is generated from this file.
- **`Debug/ti_msp_dl_config.h` / `ti_msp_dl_config.c`** — Auto-generated by SysConfig. **Do not edit.** Provides `SYSCFG_DL_init()`, pin macros (`LED_PORT`, `LED_PIN_0_PIN`), and clock config (CPUCLK at 32 MHz).
- **`Debug/TEST.out`** — Final ELF executable for the MSPM0G3507.
- **`Debug/TEST.map`** — Linker map file.

## Key Dependencies

- **MSPM0 SDK** v2.10.00.04 — DriverLib, CMSIS headers, linker scripts
- **SysConfig** v1.26.2 — Code generation tool for pin mux/peripheral configuration
- **TI ARM Clang** v4.0.4.LTS — Compiler (not GCC)

## Pin Assignments

| Pin  | Function     | Notes                          |
| PA0  | GPIO Output  | LED                            |
| PA19 | SWDIO        | Debug data (J101 13:14 ON)     |
| PA20 | SWCLK        | Debug clock (J101 15:16 ON)    |

## SysConfig

Peripheral configuration is done through `empty.syscfg` in the SysConfig GUI. Changing this file regenerates `Debug/ti_msp_dl_config.c`/`.h`. The `.syscfg` file targets device "MSPM0G3507" in package "LQFP-48(PT)".
